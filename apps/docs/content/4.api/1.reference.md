# API Reference

## Components

### `<FlowNarrator>`

The top-level loader. It accepts a single spec, a manifest, or a URL to either.

```vue
<FlowNarrator src="./flow-specs/manifest.json" :auto-play="true" :interval="3000" />
```

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `src` | `FlowNarratorSource` | — | Required. A spec object, manifest object, or JSON path/URL string. |
| `autoPlay` | `boolean` | `false` | Start playback automatically on load. |
| `interval` | `number` | `3000` | Milliseconds between auto-advance steps. |

`FlowNarratorSource` is `FlowNarratorSpec | FlowNarratorManifest | string`.

Emitted events:

| Event | Payload | Description |
| ----- | ------- | ----------- |
| `loaded` | `FlowNarratorSpec` | Fires after a spec is resolved and normalized. |
| `load-error` | `string` | Fires when a spec or manifest cannot be loaded. |
| `state-change` | `Array<{ path: string; value: unknown }>` | Fires when the internal state model changes. |

### `<FlowNarratorPanel>`

Fixed-height wrapper around `<FlowNarrator>` with a built-in fullscreen toggle. This is what the docs site and Slidev example use.

```vue
<FlowNarratorPanel
  :src="spec"
  height="min(72vh, 760px)"
  :min-height="420"
  :fullscreen-enabled="true"
/>
```

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `src` | `FlowNarratorSource` | — | Required. Same source contract as `<FlowNarrator>`. |
| `autoPlay` | `boolean` | `false` | Start playback automatically on load. |
| `interval` | `number` | `3000` | Milliseconds between auto-advance steps. |
| `height` | `number \| string` | `"min(72vh, 760px)"` | Panel height. |
| `minHeight` | `number` | `420` | Minimum panel height in px. |
| `zIndex` | `number` | `1400` | Fullscreen overlay z-index. |
| `fullscreenEnabled` | `boolean` | `true` | Show or hide the fullscreen toggle. |

## Nuxt Module

`@flow-narrator/nuxt` adds CSS, global wrapper components, and a small prepare endpoint used to resolve relative manifest/spec URLs safely in SSR and content-driven embeds.

```ts
export default defineNuxtConfig({
  modules: ["@flow-narrator/nuxt"],
});
```

Module options:

| Option | Type | Default | Description |
| ------ | ---- | ------- | ----------- |
| `css` | `boolean` | `true` | Automatically inject `flow-narrator/style.css`. |
| `components` | `boolean` | `true` | Auto-register the Nuxt wrapper components globally. |

## Composables

### `useFlowNarratorRuntime()`

Access the resolved spec and manifest selection state from inside a mounted `<FlowNarrator>` tree.

```ts
import { useFlowNarratorRuntime } from "flow-narrator";

const { spec, flowOptions, activeFlowId, setActiveFlow, interval } = useFlowNarratorRuntime();
```

| Return | Type | Description |
| ------ | ---- | ----------- |
| `spec` | `Ref<FlowNarratorSpec \| null>` | Currently resolved spec. |
| `interval` | `Ref<number>` | Active autoplay interval. |
| `flowOptions` | `Ref<FlowNarratorFlowOption[]>` | Manifest-derived flow picker options. |
| `activeFlowId` | `Ref<string \| null>` | Currently selected flow id. |
| `setActiveFlow()` | `(flowId: string) => void` | Switch to another flow in the current manifest. |

### `useFlowNarratorFullscreen()`

Low-level helper used by `FlowNarratorPanel` to manage fullscreen state and `Escape` handling.

```ts
import { useFlowNarratorFullscreen } from "flow-narrator";

const { fullscreen, toggleFullscreen, closeFullscreen } = useFlowNarratorFullscreen();
```

| Return | Type | Description |
| ------ | ---- | ----------- |
| `fullscreen` | `Ref<boolean>` | Whether fullscreen mode is active. |
| `toggleFullscreen()` | `() => void` | Toggle fullscreen on or off. |
| `closeFullscreen()` | `() => void` | Force fullscreen off. |

### `useShiki(langs?, theme?)`

Shared singleton Shiki highlighter used by code nodes.

```ts
import { useShiki } from "flow-narrator";

const { highlighter, ready } = useShiki();
await ready;

const html = highlighter.value?.codeToHtml("const x = 1", {
  lang: "typescript",
  theme: "github-dark",
});
```

| Return | Type | Description |
| ------ | ---- | ----------- |
| `highlighter` | `ShallowRef<HighlighterCore \| null>` | Shared Shiki instance once loaded. |
| `ready` | `Promise<HighlighterCore>` | Promise that resolves when the highlighter is ready. |

### `useTimeline({ totalSteps, interval? })`

Low-level timeline helper for building custom playback controls.

```ts
import { ref } from "vue";
import { useTimeline } from "flow-narrator";

const totalSteps = ref(5);
const { currentStep, playing, progress, next, prev, goTo, togglePlay } = useTimeline({
  totalSteps,
  interval: 2500,
});
```

| Return | Type | Description |
| ------ | ---- | ----------- |
| `currentStep` | `Ref<number>` | Active step index. |
| `playing` | `Ref<boolean>` | Whether autoplay is running. |
| `totalSteps` | `Ref<number>` | Total step count passed in. |
| `progress` | `ComputedRef<number>` | Normalized progress from `0` to `1`. |
| `next()` | `() => void` | Advance one step. |
| `prev()` | `() => void` | Move back one step. |
| `goTo()` | `(step: number) => void` | Jump to a specific step. |
| `togglePlay()` | `() => void` | Toggle autoplay. |

## Twoslash Helpers

These helpers are exported from `flow-narrator` for authoring and runtime checks around twoslash-enabled code nodes:

- `hasTwoslashHints(code)`
- `hasTwoslashHtml(value)`
- `normalizeTwoslashHtml(value)`
- `normalizeTwoslashLanguage(language)`
- `resolveTwoslashHtmlForTheme(value, theme)`

## Catalog & Registry

flow-narrator exposes the json-render catalog and registry for advanced programmatic use.

```ts
import { catalog, registry } from "flow-narrator";
```

- `catalog` describes the flow-narrator schema and component props.
- `registry` maps schema component names to Vue components.
