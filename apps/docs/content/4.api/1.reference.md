# API Reference

## Components

### `<Flaier>`

The top-level loader. It accepts a single spec, a manifest, or a URL to either.

```vue
<Flaier src="./flow-specs/manifest.json" :auto-play="true" :interval="3000" />
```

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `src` | `FlaierSource` | — | Required. A spec object, manifest object, or JSON path/URL string. |
| `autoPlay` | `boolean` | `false` | Start playback automatically on load. |
| `interval` | `number` | `3000` | Milliseconds between auto-advance steps. |
| `nodes` | `FlaierCustomNodeDefinitions` | — | Register additional flow node types for this renderer instance. |

`FlaierSource` is `FlaierSpec | FlaierManifest | string`.

Emitted events:

| Event | Payload | Description |
| ----- | ------- | ----------- |
| `loaded` | `FlaierSpec` | Fires after a spec is resolved and normalized. |
| `load-error` | `string` | Fires when a spec or manifest cannot be loaded. |
| `state-change` | `Array<{ path: string; value: unknown }>` | Fires when the internal state model changes. |

### `<FlaierPanel>`

Fixed-height wrapper around `<Flaier>` with a built-in fullscreen toggle. This is what the docs site and Slidev example use.

```vue
<FlaierPanel
  :src="spec"
  height="min(72vh, 760px)"
  :min-height="420"
  :fullscreen-enabled="true"
/>
```

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `src` | `FlaierSource` | — | Required. Same source contract as `<Flaier>`. |
| `autoPlay` | `boolean` | `false` | Start playback automatically on load. |
| `interval` | `number` | `3000` | Milliseconds between auto-advance steps. |
| `nodes` | `FlaierCustomNodeDefinitions` | — | Same custom node contract as `<Flaier>`. |
| `height` | `number \| string` | `"min(72vh, 760px)"` | Panel height. |
| `minHeight` | `number` | `420` | Minimum panel height in px. |
| `zIndex` | `number` | `1400` | Fullscreen overlay z-index. |
| `fullscreenEnabled` | `boolean` | `true` | Show or hide the fullscreen toggle. |

## Nuxt Module

`@flaier/nuxt` adds CSS, global wrapper components, and a small prepare endpoint used to resolve relative manifest/spec URLs safely in SSR and content-driven embeds.

```ts
export default defineNuxtConfig({
  modules: ["@flaier/nuxt"],
});
```

Module options:

| Option | Type | Default | Description |
| ------ | ---- | ------- | ----------- |
| `css` | `boolean` | `true` | Automatically inject `@flaier/core/style.css`. |
| `components` | `boolean` | `true` | Auto-register the Nuxt wrapper components globally. |

### `registerFlaierNodes(nodes)`

Nuxt auto-import that registers custom node types application-wide. This is the easiest way to make custom nodes available to `FlaierMdc` inside markdown/content pages.

This only covers Flaier renderer nodes. If you render your own wrapper component directly from markdown, like `<CustomStyledFlowDemo />`, that Vue component still needs to be registered globally by your app.

```ts
export default defineNuxtPlugin(() => {
  registerFlaierNodes({
    CalloutNode: {
      props: z.object({
        label: z.string(),
        tone: z.enum(["info", "warning"]).default("info"),
      }),
      description: "Compact callout card for narrative notes",
      component: CalloutNode,
    },
  });
});
```

## Composables

### `useFlaierRuntime()`

Access the resolved spec and manifest selection state from inside a mounted `<Flaier>` tree.

```ts
import { useFlaierRuntime } from "@flaier/core";

const { spec, flowOptions, activeFlowId, setActiveFlow, interval } = useFlaierRuntime();
```

| Return | Type | Description |
| ------ | ---- | ----------- |
| `spec` | `Ref<FlaierSpec \| null>` | Currently resolved spec. |
| `interval` | `Ref<number>` | Active autoplay interval. |
| `flowOptions` | `Ref<FlaierFlowOption[]>` | Manifest-derived flow picker options. |
| `activeFlowId` | `Ref<string \| null>` | Currently selected flow id. |
| `setActiveFlow()` | `(flowId: string) => void` | Switch to another flow in the current manifest. |

### `useFlaierFullscreen()`

Low-level helper used by `FlaierPanel` to manage fullscreen state and `Escape` handling.

```ts
import { useFlaierFullscreen } from "@flaier/core";

const { fullscreen, toggleFullscreen, closeFullscreen } = useFlaierFullscreen();
```

| Return | Type | Description |
| ------ | ---- | ----------- |
| `fullscreen` | `Ref<boolean>` | Whether fullscreen mode is active. |
| `toggleFullscreen()` | `() => void` | Toggle fullscreen on or off. |
| `closeFullscreen()` | `() => void` | Force fullscreen off. |

### `useShiki(langs?, theme?)`

Shared singleton Shiki highlighter used by code nodes.

```ts
import { useShiki } from "@flaier/core";

const { highlighter, ready } = useShiki();
await ready;

const html = highlighter.value?.codeToHtml("const x = 1", {
  lang: "typescript",
  theme: "github-dark",
});
```

| Return | Type | Description |
| ------ | ---- | ----------- |
| `highlighter` | `ShallowRef<HighlighterCore \| null>` | Shared Shiki instance once loaded. |
| `ready` | `Promise<HighlighterCore>` | Promise that resolves when the highlighter is ready. |

### `useTimeline({ totalSteps, interval? })`

Low-level timeline helper for building custom playback controls.

```ts
import { ref } from "vue";
import { useTimeline } from "@flaier/core";

const totalSteps = ref(5);
const { currentStep, playing, progress, next, prev, goTo, togglePlay } = useTimeline({
  totalSteps,
  interval: 2500,
});
```

| Return | Type | Description |
| ------ | ---- | ----------- |
| `currentStep` | `Ref<number>` | Active step index. |
| `playing` | `Ref<boolean>` | Whether autoplay is running. |
| `totalSteps` | `Ref<number>` | Total step count passed in. |
| `progress` | `ComputedRef<number>` | Normalized progress from `0` to `1`. |
| `next()` | `() => void` | Advance one step. |
| `prev()` | `() => void` | Move back one step. |
| `goTo()` | `(step: number) => void` | Jump to a specific step. |
| `togglePlay()` | `() => void` | Toggle autoplay. |

## Catalog & Registry

flaier exposes the json-render catalog and registry for advanced programmatic use.

```ts
import {
  catalog,
  createFlaierCatalog,
  createFlaierRegistry,
  edgeTransitionSchema,
  registry,
  sourceAnchorSchema,
} from "@flaier/core";
```

- `catalog` describes the flaier schema and component props.
- `registry` maps schema component names to Vue components.
- `createFlaierCatalog({ nodes })` merges your custom nodes into the AI-facing catalog so `catalog.prompt()` includes them.
- `createFlaierRegistry({ nodes })` returns a matching renderer registry for advanced programmatic use.
- `sourceAnchorSchema` and `edgeTransitionSchema` are exported so custom node props can reuse the built-in linking/transition shapes.

Example:

```ts
const nodes = {
  CalloutNode: {
    props: z.object({
      label: z.string(),
      body: z.string(),
      sourceAnchor: sourceAnchorSchema.optional(),
      transitions: z.array(edgeTransitionSchema).optional(),
    }),
    description: "Narrative callout block rendered inside a flow",
    component: CalloutNode,
  },
};

const customCatalog = createFlaierCatalog({ nodes });
const { registry: customRegistry } = createFlaierRegistry({ nodes });
```

## Styling

The default stylesheet is token-based. Override CSS variables on `.flaier` or on your surrounding panel wrapper to change colors and chrome without replacing the renderer.

Stable hooks exposed by the renderer:

- `.flaier[data-mode="narrative" | "architecture"]`
- `.flaier[data-theme="dark" | "light"]`
- `.flaier[data-focus-mode="focus" | "overview"]`
- `.fn-node[data-node-type]`
- `.fn-node[data-node-kind]`
- `.fn-node[data-node-status]`

Panel/embed chrome is also tokenized through CSS variables such as `--fn-panel-toggle-bg`, `--fn-panel-toggle-border`, and `--fn-panel-backdrop-bg`.
