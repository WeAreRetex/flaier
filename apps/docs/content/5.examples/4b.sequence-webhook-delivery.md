# Sequence Webhook Delivery

This example leans into the parts of sequence mode that are hard to express cleanly in a DAG: async acknowledgement, queue handoff, nested retry groups, and a worker self-message for backoff calculation.

**Spec:** `sequence-webhook.flow.json` · **Mode:** sequence · **Participants:** 5 · **Features:** nested `alt`, `loop`, `opt`, self-message

## What it demonstrates

- Async intake where the provider gets a fast `202 Accepted`
- Queue-to-worker handoff after the external request is already finished
- Nested `alt` inside a `loop` for delivery success vs retryable failure
- Worker self-message for exponential backoff scheduling
- `opt` escalation path to on-call after repeated failures

## When to use

Sequence webhook flows work well for:

- Webhook delivery pipelines
- Job dispatch and retry orchestration
- External integration contracts
- Incident and runbook documentation for async systems
