# Sequence Auth Flow

This example uses the new `sequence` mode to model an auth login as participant lifelines instead of DAG nodes. It shows how Flaier can keep a familiar sequence-diagram shape while still giving you playback and rich step descriptions.

**Spec:** `sequence.flow.json` · **Mode:** sequence · **Participants:** 4 · **Features:** activations, `alt`, `loop`, `opt`

## What it demonstrates

- Explicit participant ordering with `FlowTimeline.props.participants`
- Sync and return messages across client, gateway, auth service, and database
- `alt` branches for valid vs invalid credentials
- `loop` for silent session refresh
- `opt` for suspicious-login auditing
- Notes attached to one or two participants

## When to use

Sequence auth flows work well for:

- Login and session refresh handshakes
- OAuth callback exchanges
- Token issuance and rotation walkthroughs
- Security review docs where request order matters
