# CLI Commands

All commands are run from the repo root via `vp run`.

## scaffold

Generate a starter flow spec from a template.

```bash
vp run agents:scaffold -- --title "Checkout Flow" --template branching --out ./flow-specs/checkout.flow.json
```

### Options

| Flag                   | Description                              | Default                         |
| ---------------------- | ---------------------------------------- | ------------------------------- |
| `--title <text>`       | Flow title (required)                    | —                               |
| `--description <text>` | Flow subtitle                            | —                               |
| `--template <name>`    | `linear`, `branching`, or `architecture` | `linear`                        |
| `--language <name>`    | CodeNode syntax language                 | `typescript`                    |
| `--out <path>`         | Output file path                         | `./flow-specs/<slug>.flow.json` |
| `--force`              | Overwrite existing file                  | —                               |

### Examples

```bash
# Linear flow with a custom output path
vp run agents:scaffold -- --title "User Registration" --out ./flow-specs/user-reg.flow.json

# Branching flow for a validation step
vp run agents:scaffold -- --title "Webhook Dispatch" --template branching --language typescript

# Architecture diagram
vp run agents:scaffold -- --title "Payment System" --template architecture --out ./flow-specs/payment-arch.flow.json
```

## validate

Validate a spec or manifest against the schema and flow-narrator readiness rules.

```bash
vp run agents:validate -- ./flow-specs/manifest.json
```

Validation checks:

- JSON schema compliance (`@json-render/core` schema)
- All `children` IDs reference existing elements
- All `transitions[].to` IDs reference existing elements
- `FlowTimeline` is the root element
- Required props are present on each node type

### Options

| Flag     | Description |
| -------- | ----------- |
| `--help` | Show usage  |

## manifest

Scan a directory for `*.flow.json` files and write a manifest.

```bash
vp run agents:manifest -- --dir ./flow-specs --out ./flow-specs/manifest.json
```

### Options

| Flag           | Description          | Default |
| -------------- | -------------------- | ------- |
| `--dir <path>` | Directory to scan    | —       |
| `--out <path>` | Output manifest path | —       |

The generated manifest derives each flow's `id` from the filename (without `.flow.json`) and `title` from the spec's `FlowTimeline.props.title`. Existing manifest entries are preserved; new files are appended.
