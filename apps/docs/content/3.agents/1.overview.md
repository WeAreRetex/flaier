# Agents Overview

The `packages/agents` package provides CLI tools designed for AI coding agents (Claude, Cursor, etc.) to scaffold, validate, and package flow specs without manual editing.

## The AI generation loop

1. **Scaffold** — Generate a starter spec from a template.
2. **Implement** — The AI agent fills in real node props (labels, code, transitions, etc.).
3. **Validate** — Check the spec passes schema and flow-narrator readiness rules.
4. **Manifest** — Register the spec in the manifest index.
5. **Render** — Open the viewer to verify the output visually.

## Claude skill integration

The agents package ships as a Claude Code skill. When installed, Claude can scaffold and validate specs directly from the chat:

```bash
pnpm agents:scaffold -- --title "Checkout Flow" --template branching --out ./apps/viewer/flow-specs/checkout.flow.json
pnpm agents:validate -- ./apps/viewer/flow-specs/checkout.flow.json
pnpm agents:manifest -- --dir ./apps/viewer/flow-specs --out ./apps/viewer/flow-specs/manifest.json
```

In this repo, viewer-ready specs typically live in `apps/viewer/flow-specs`, while presentation-oriented architecture specs can live in `apps/slides/flow-specs`.

## Templates

Three scaffold templates are available:

| Template       | Description                                                              |
| -------------- | ------------------------------------------------------------------------ |
| `linear`       | A straight-line flow: trigger → code → description → link.               |
| `branching`    | A decision tree: trigger → decision → success path + error path.         |
| `architecture` | A full architecture diagram with zones, services, queues, and databases. |

Start with the template that best matches your flow shape, then replace the placeholder nodes with real content.

## Incremental generation

AI harnesses can generate each `*.flow.json` in separate sessions and append entries to the manifest incrementally. The manifest format is designed for this — each flow entry is independent, and `defaultFlowId` can be updated to point to the newest flow.

This means large codebases can be narrated piece by piece without loading the full codebase into a single context window.
