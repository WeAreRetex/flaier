# Rendering Modes

Set `mode` on the `FlowTimeline` props (or pass it as a top-level prop on `<Flaier>`).

```json
{
  "type": "FlowTimeline",
  "props": {
    "title": "Auth Login",
    "mode": "narrative"
  }
}
```

## Narrative mode (default)

`mode: "narrative"` renders the flow as a **step-by-step walkthrough**:

- Timeline controls at the bottom (prev / next / play-pause)
- Active node is highlighted and centered
- Autoplay advances through nodes at the configured `interval`
- Ideal for code walkthroughs, onboarding flows, and incident post-mortems

```vue
<Flaier src="./login.flow.json" :interval="3000" :auto-play="true" />
```

## Architecture mode

`mode: "architecture"` renders the flow as a **static topology map**:

- All nodes visible simultaneously
- Click a node to open its inspector panel with full metadata
- Zone boundaries drawn around grouped nodes
- Export controls (PNG / PDF) in the top-right corner
- Ideal for system design documents, architecture reviews, and runbooks

```json
{
  "type": "FlowTimeline",
  "props": {
    "title": "Payment System",
    "mode": "architecture",
    "zones": [
      { "id": "edge", "label": "Edge", "color": "#6366f1", "padding": 64 },
      { "id": "core", "label": "Core Services", "color": "#0ea5e9", "padding": 68 }
    ]
  }
}
```

### Zones

Zones group `ArchitectureNode` entries visually by setting `zone` on node props to match a zone `id`.

```json
{
  "type": "ArchitectureNode",
  "props": {
    "label": "API Service",
    "kind": "service",
    "zone": "core"
  }
}
```

See [Node Types → ArchitectureNode](/guide/node-types#architecturenode) for the full set of architecture-specific props.

## Switching mode at runtime

The `<Flaier>` component re-renders when `src` changes, so you can swap between a narrative spec and an architecture spec dynamically.

To toggle mode inside a single spec, update the `FlowTimeline.props.mode` field and reload the spec.

## Embedded docs and slides

When architecture mode is embedded in docs pages or slide decks, it often needs less chrome and tighter theme integration than a standalone viewer.

```json
{
  "type": "FlowTimeline",
  "props": {
    "mode": "architecture",
    "themeMode": "document",
    "showHeaderOverlay": false,
    "showExportControls": false,
    "showThemeToggle": true,
    "showArchitectureInspector": true,
    "defaultArchitectureInspectorOpen": false,
    "showArchitectureInspectorToggleText": false
  }
}
```

That is the pattern used by `apps/slides/flow-specs/story-architecture.flow.json` in this repo's Slidev example.
