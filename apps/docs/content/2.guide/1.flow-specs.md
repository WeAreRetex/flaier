# Flow Specs

A flow spec is a JSON file with three top-level keys: `root`, `elements`, and optionally `state`.

```json
{
  "root": "timeline",
  "state": { "currentStep": 0, "playing": false },
  "elements": {
    "timeline": { ... },
    "trigger-entry": { ... },
    "code-main": { ... }
  }
}
```

## `root`

The ID of the top-level element. Always `"timeline"`.

## `elements`

A flat map of element IDs to element definitions. Each element has:

| Key        | Type       | Description                                                     |
| ---------- | ---------- | --------------------------------------------------------------- |
| `type`     | `string`   | The node type — `FlowTimeline`, `TriggerNode`, `CodeNode`, etc. |
| `props`    | `object`   | Type-specific props. See [Node Types](/guide/node-types).       |
| `children` | `string[]` | IDs of child elements (defines graph edges).                    |

The graph is encoded as parent → child ID references — there is no nesting. This makes it easy for LLMs to generate incrementally.

## `FlowTimeline` element

Every spec must have exactly one `FlowTimeline` element as the root.

```json
{
  "type": "FlowTimeline",
  "props": {
    "title": "Auth Login Flow",
    "description": "From request to session issuance",
    "mode": "narrative",
    "minHeight": 620,
    "layoutEngine": "dagre",
    "layoutRankSep": 260,
    "layoutNodeSep": 170,
    "layoutEdgeSep": 34
  },
  "children": ["trigger-entry"]
}
```

| Prop            | Type                            | Default        | Description                        |
| --------------- | ------------------------------- | -------------- | ---------------------------------- |
| `title`         | `string`                        | —              | Required. Displayed in the header. |
| `description`   | `string`                        | —              | Subtitle shown below the title.    |
| `mode`          | `"narrative" \| "architecture"` | `"narrative"`  | Rendering mode.                    |
| `direction`     | `"horizontal" \| "vertical"`    | `"horizontal"` | Layout direction.                  |
| `minHeight`     | `number`                        | `620`          | Canvas minimum height in px.       |
| `layoutEngine`  | `"dagre" \| "manual"`           | `"dagre"`      | Auto-layout or manual positioning. |
| `layoutRankSep` | `number`                        | `260`          | Dagre rank separation.             |
| `layoutNodeSep` | `number`                        | `170`          | Dagre node separation.             |
| `layoutEdgeSep` | `number`                        | `34`           | Dagre edge separation.             |
| `zones`         | `ArchitectureZone[]`            | —              | Zone groups for architecture mode. |

## `transitions` on nodes

Each node can declare its outgoing edges via `transitions` on its `props`. This gives edges semantic metadata beyond simple parent → child wiring.

```json
{
  "transitions": [
    {
      "to": "code-commit",
      "label": "Valid payload",
      "kind": "success",
      "protocol": "HTTPS",
      "transport": "sync",
      "criticality": "high"
    }
  ]
}
```

| Field         | Type                 | Values                                                     |
| ------------- | -------------------- | ---------------------------------------------------------- |
| `to`          | `string`             | Target element ID.                                         |
| `label`       | `string`             | Edge label shown on the diagram.                           |
| `kind`        | `EdgeTransitionKind` | `default`, `success`, `error`, `warning`, `retry`, `async` |
| `protocol`    | `string`             | e.g. `HTTPS`, `Kafka`, `SQL`                               |
| `transport`   | `string`             | `sync` or `async`                                          |
| `auth`        | `string`             | Auth mechanism on this edge.                               |
| `contract`    | `string`             | API contract or schema version.                            |
| `criticality` | `string`             | `low`, `medium`, `high`                                    |

## Minimal example

```json
{
  "root": "timeline",
  "elements": {
    "timeline": {
      "type": "FlowTimeline",
      "props": { "title": "Hello Flow" },
      "children": ["start"]
    },
    "start": {
      "type": "TriggerNode",
      "props": { "label": "HTTP POST /login" }
    }
  }
}
```
