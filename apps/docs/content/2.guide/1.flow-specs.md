# Flow Specs

A flow spec is a JSON file with three top-level keys: `root`, `elements`, and optionally `state`.

```json
{
  "root": "timeline",
  "state": { "currentStep": 0, "playing": false },
  "elements": {
    "timeline": { ... },
    "trigger-entry": { ... },
    "code-main": { ... }
  }
}
```

## `root`

The ID of the top-level element. It must point to the single `FlowTimeline` root element. `"timeline"` is the common convention, but any key is valid.

## `elements`

A flat map of element IDs to element definitions. Each element has:

| Key        | Type       | Description                                                     |
| ---------- | ---------- | --------------------------------------------------------------- |
| `type`     | `string`   | The node type — `FlowTimeline`, `TriggerNode`, `CodeNode`, etc. |
| `props`    | `object`   | Type-specific props. See [Node Types](/guide/node-types).       |
| `children` | `string[]` | IDs of child elements (defines graph edges).                    |

For narrative and architecture flows, the graph is encoded as parent → child ID references. In `mode: "sequence"`, `children` become an ordered statement list and `FlowTimeline.props.participants` controls lifeline order.

## `FlowTimeline` element

Every spec must have exactly one `FlowTimeline` element as the root.

```json
{
  "type": "FlowTimeline",
  "props": {
    "title": "Auth Login Flow",
    "description": "From request to session issuance",
    "mode": "narrative",
    "minHeight": 620,
    "layoutEngine": "dagre",
    "layoutRankSep": 260,
    "layoutNodeSep": 170,
    "layoutEdgeSep": 34
  },
  "children": ["trigger-entry"]
}
```

| Prop            | Type                            | Default        | Description                        |
| --------------- | ------------------------------- | -------------- | ---------------------------------- |
| `title`         | `string`                        | —              | Required. Displayed in the header. |
| `description`   | `string`                        | —              | Subtitle shown below the title.    |
| `mode`          | `"narrative" \| "sequence" \| "architecture"` | `"narrative"`  | Rendering mode.                    |
| `participants`  | `string[]`                      | —              | Sequence participant order for `mode: "sequence"`. |
| `showSequenceNumbers` | `boolean`                | `false`        | Show step numbers on sequence messages and notes. |
| `direction`     | `"horizontal" \| "vertical"`    | `"horizontal"` | Layout direction.                  |
| `minHeight`     | `number`                        | `620`          | Canvas minimum height in px.       |
| `layoutEngine`  | `"dagre" \| "manual"`           | `"dagre"`      | Auto-layout or manual positioning. |
| `layoutRankSep` | `number`                        | `260`          | Dagre rank separation.             |
| `layoutNodeSep` | `number`                        | `170`          | Dagre node separation.             |
| `layoutEdgeSep` | `number`                        | `34`           | Dagre edge separation.             |
| `zones`         | `ArchitectureZone[]`            | —              | Zone groups for architecture mode. |

### Embedding and presentation props

These extra `FlowTimeline.props` fields are useful when embedding the renderer inside docs pages or slide decks:

| Prop                                   | Type                    | Default      | Description |
| -------------------------------------- | ----------------------- | ------------ | ----------- |
| `themeMode`                            | `"local" \| "document"` | `"local"`   | Use the component's own theme toggle, or inherit the surrounding document theme. |
| `showHeaderOverlay`                    | `boolean`               | `true`       | Show or hide the floating title + manifest picker overlay. |
| `showExportControls`                   | `boolean`               | `true`       | Show or hide the architecture export button. |
| `showThemeToggle`                      | `boolean`               | `true`       | Show or hide the light/dark theme toggle. |
| `showArchitectureInspector`            | `boolean`               | `true`       | Enable the architecture inspector side panel. |
| `defaultArchitectureInspectorOpen`     | `boolean`               | `true`       | Open the architecture inspector by default. |
| `showArchitectureInspectorToggleText`  | `boolean`               | `true`       | Show the text label next to the inspector toggle icon. |

The Slidev example in this repo uses `themeMode: "document"`, hides the header/export chrome, and starts with the inspector collapsed so the diagram fits inside a deck canvas.

## `transitions` on nodes

Each node can declare its outgoing edges via `transitions` on its `props`. This gives edges semantic metadata beyond simple parent → child wiring.

```json
{
  "transitions": [
    {
      "to": "code-commit",
      "label": "Valid payload",
      "kind": "success",
      "protocol": "HTTPS",
      "transport": "sync",
      "criticality": "high"
    }
  ]
}
```

| Field         | Type                 | Values                                                     |
| ------------- | -------------------- | ---------------------------------------------------------- |
| `to`          | `string`             | Target element ID.                                         |
| `label`       | `string`             | Edge label shown on the diagram.                           |
| `kind`        | `EdgeTransitionKind` | `default`, `success`, `error`, `warning`, `retry`, `async` |
| `protocol`    | `string`             | e.g. `HTTPS`, `Kafka`, `SQL`                               |
| `transport`   | `string`             | `sync` or `async`                                          |
| `auth`        | `string`             | Auth mechanism on this edge.                               |
| `contract`    | `string`             | API contract or schema version.                            |
| `criticality` | `string`             | `low`, `medium`, `high`                                    |

## Sequence mode structure

Sequence specs use the same flat `elements` map, but the semantics are different from DAG-based flows:

- `FlowTimeline.props.participants` declares the visible lifeline order.
- Each id in `participants` must point to a `SequenceParticipant` element.
- `FlowTimeline.children` lists top-level ordered statements.
- Top-level sequence statements are usually `SequenceMessage`, `SequenceNote`, or `SequenceGroup`.
- `SequenceGroup.children` should point to `SequenceBranch` elements.
- `SequenceBranch.children` then contain the ordered statements inside that branch.

Minimal sequence root:

```json
{
  "type": "FlowTimeline",
  "props": {
    "title": "Checkout Handshake",
    "mode": "sequence",
    "participants": ["browser", "api", "payments"]
  },
  "children": ["msg-start", "group-payment-result"]
}
```

## Minimal example

```json
{
  "root": "timeline",
  "elements": {
    "timeline": {
      "type": "FlowTimeline",
      "props": { "title": "Hello Flow" },
      "children": ["start"]
    },
    "start": {
      "type": "TriggerNode",
      "props": { "label": "HTTP POST /login" }
    }
  }
}
```
