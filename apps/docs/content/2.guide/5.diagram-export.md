# Diagram Export

In architecture mode, an export button appears in the top-right corner of the renderer. Click it to download the **full diagram** — not just the visible viewport.

## Supported formats

| Format  | Notes                                                               |
| ------- | ------------------------------------------------------------------- |
| **PNG** | High-resolution raster image. Good for presentations and documents. |
| **PDF** | Vector-based. Good for technical documentation and printing.        |

The export captures all laid-out nodes and edges, including those outside the current scroll position. The output dimensions match the full canvas size.

## How it works

Export is powered by `html-to-image` and `jspdf`. The renderer expands to its full content size, captures the canvas, then restores the viewport. This happens in a single frame — no visible flash.

## Programmatic export

You can trigger export from your own code using the `useFlowNarratorRuntime` composable:

```vue
<script setup lang="ts">
import { useFlowNarratorRuntime } from "flow-narrator";

const { exportPng, exportPdf } = useFlowNarratorRuntime();
</script>

<template>
  <button @click="exportPng()">Export PNG</button>
  <button @click="exportPdf()">Export PDF</button>
</template>
```

## Tips

- Use `mode: "architecture"` for export — narrative mode only shows the active node, which produces a sparse diagram.
- Set `minHeight` on `FlowTimeline` to ensure the canvas is tall enough to fit all nodes without overlap.
- For large diagrams, increase `layoutRankSep` and `layoutNodeSep` to give nodes more breathing room.
