# Diagram Export

In architecture mode, an export button appears in the top-right corner of the renderer. Click it to download the **full diagram** — not just the visible viewport.

## Supported formats

| Format  | Notes                                                               |
| ------- | ------------------------------------------------------------------- |
| **PNG** | High-resolution raster image. Good for presentations and documents. |
| **PDF** | Printable PDF generated from the full captured diagram image.       |

The export captures all laid-out nodes and edges, including those outside the current scroll position. The output dimensions match the full canvas size.

## How it works

Export is powered by `html-to-image` and `jspdf`. The renderer clones the full diagram into an off-screen capture stage, renders it as a high-resolution image, and then downloads either that PNG directly or the same capture inside a PDF.

## Built-in export controls

Export is currently exposed through the renderer's built-in architecture controls. If you want export available to end users, keep `showExportControls` enabled on the `FlowTimeline` props.

## Tips

- Use `mode: "architecture"` for export — narrative mode only shows the active node, which produces a sparse diagram.
- Set `minHeight` on `FlowTimeline` to ensure the canvas is tall enough to fit all nodes without overlap.
- For large diagrams, increase `layoutRankSep` and `layoutNodeSep` to give nodes more breathing room.
- Set `showExportControls: false` when the same spec is embedded in a constrained docs page or slide deck and you do not want export UI visible.
