# Node Types

Every element in a spec has a `type` that maps to one of these node types. Each type renders differently and accepts a different set of `props`.

## TriggerNode

Entry points — HTTP routes, cron jobs, webhooks, queue consumers.

```json
{
  "type": "TriggerNode",
  "props": {
    "label": "POST /api/login",
    "description": "Accepts email + password",
    "color": "#22c55e",
    "sourceAnchor": { "path": "src/api/login.ts", "line": 12 }
  },
  "children": ["code-validate"]
}
```

| Prop           | Type                | Description                 |
| -------------- | ------------------- | --------------------------- |
| `label`        | `string`            | Required. Node title.       |
| `description`  | `string`            | Subtitle shown below label. |
| `color`        | `string`            | Accent color (hex or CSS).  |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file.        |
| `transitions`  | `EdgeTransition[]`  | Outgoing edge metadata.     |

## CodeNode

Source code steps with syntax highlighting, magic-move animation, and optional twoslash type hints.

```json
{
  "type": "CodeNode",
  "props": {
    "label": "Validate credentials",
    "file": "src/auth/validate.ts",
    "language": "typescript",
    "code": "export async function validate(email: string, password: string) {\n  const user = await db.user.findByEmail(email)\n  return verify(password, user.passwordHash)\n}",
    "comment": "Constant-time comparison prevents timing attacks.",
    "story": "On success, returns the user record. On failure, throws AuthError."
  }
}
```

| Prop             | Type                | Description                                      |
| ---------------- | ------------------- | ------------------------------------------------ |
| `label`          | `string`            | Required. Node title.                            |
| `code`           | `string`            | Required. The code to display.                   |
| `language`       | `string`            | Shiki language id (e.g. `typescript`).           |
| `file`           | `string`            | File path shown in the node header.              |
| `comment`        | `string`            | Technical annotation about the code.             |
| `story`          | `string`            | Narrative explanation for non-technical readers. |
| `wrapLongLines`  | `boolean`           | Wrap long lines instead of scrolling.            |
| `twoslash`       | `boolean`           | Enable TypeScript type hints via twoslash.       |
| `magicMoveSteps` | `MagicMoveStep[]`   | Animated code evolution steps.                   |
| `sourceAnchor`   | `SourceAnchorInput` | Link to source file.                             |
| `transitions`    | `EdgeTransition[]`  | Outgoing edge metadata.                          |

### Magic-move steps

Animate code evolution through multiple versions with smooth transitions.

```json
{
  "magicMoveSteps": [
    { "code": "const x = 1", "title": "Initial", "comment": "Before refactor" },
    { "code": "const x = computeValue()", "title": "After", "story": "Extracted to a function" }
  ]
}
```

## DecisionNode

Branch points — conditional logic, routing, feature flags.

```json
{
  "type": "DecisionNode",
  "props": {
    "label": "Is payload valid?",
    "condition": "payload.isValid === true",
    "transitions": [
      { "to": "code-commit", "label": "Valid", "kind": "success" },
      { "to": "error-invalid", "label": "Invalid", "kind": "error" }
    ]
  },
  "children": ["code-commit", "error-invalid"]
}
```

| Prop           | Type                | Description                              |
| -------------- | ------------------- | ---------------------------------------- |
| `label`        | `string`            | Required. Node title.                    |
| `condition`    | `string`            | The boolean expression being evaluated.  |
| `description`  | `string`            | Additional context.                      |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file.                     |
| `transitions`  | `EdgeTransition[]`  | Outgoing edge metadata (one per branch). |

## PayloadNode

Data snapshots showing a value before and after a transformation.

```json
{
  "type": "PayloadNode",
  "props": {
    "label": "Normalized user",
    "format": "json",
    "before": "{\"email\": \"USER@EXAMPLE.COM\"}",
    "after": "{\"email\": \"user@example.com\"}"
  }
}
```

| Prop           | Type                         | Description                             |
| -------------- | ---------------------------- | --------------------------------------- |
| `label`        | `string`                     | Required. Node title.                   |
| `payload`      | `string`                     | Single payload (no diff).               |
| `before`       | `string`                     | State before transformation.            |
| `after`        | `string`                     | State after transformation.             |
| `format`       | `"json" \| "yaml" \| "text"` | Payload format for syntax highlighting. |
| `description`  | `string`                     | Description of the transformation.      |
| `sourceAnchor` | `SourceAnchorInput`          | Link to source file.                    |
| `transitions`  | `EdgeTransition[]`           | Outgoing edge metadata.                 |

## ErrorNode

Failure paths — exceptions, error responses, fallback handling.

```json
{
  "type": "ErrorNode",
  "props": {
    "label": "Auth failed",
    "message": "Invalid credentials",
    "code": "AUTH-401",
    "cause": "Password hash mismatch",
    "mitigation": "Return 401, log failed attempt, increment rate limit counter"
  }
}
```

| Prop           | Type                | Description                    |
| -------------- | ------------------- | ------------------------------ |
| `label`        | `string`            | Required. Node title.          |
| `message`      | `string`            | Required. Error message.       |
| `code`         | `string`            | Error code.                    |
| `cause`        | `string`            | Root cause description.        |
| `mitigation`   | `string`            | Recovery or handling strategy. |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file.           |
| `transitions`  | `EdgeTransition[]`  | Outgoing edge metadata.        |

## DescriptionNode

Prose explanations — outcomes, side effects, design rationale.

```json
{
  "type": "DescriptionNode",
  "props": {
    "label": "Session issued",
    "body": "A signed JWT is returned in the response body. The session is also persisted in Redis with a 7-day TTL.",
    "sourceAnchor": "docs/auth/sessions.md:24"
  }
}
```

| Prop           | Type                | Description                                  |
| -------------- | ------------------- | -------------------------------------------- |
| `label`        | `string`            | Required. Node title.                        |
| `body`         | `string`            | Required. Prose content (supports markdown). |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file.                         |
| `transitions`  | `EdgeTransition[]`  | Outgoing edge metadata.                      |

## LinkNode

External references — source code links, dashboards, runbooks.

```json
{
  "type": "LinkNode",
  "props": {
    "label": "Auth runbook",
    "href": "https://example.com/runbooks/auth",
    "description": "Incident response steps for auth failures"
  }
}
```

| Prop           | Type                | Description                           |
| -------------- | ------------------- | ------------------------------------- |
| `label`        | `string`            | Required. Node title.                 |
| `href`         | `string`            | Required. Link URL.                   |
| `description`  | `string`            | Short description of the link target. |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file.                  |
| `transitions`  | `EdgeTransition[]`  | Outgoing edge metadata.               |

## SequenceParticipant

Declares a participant/lifeline for `mode: "sequence"`. These elements are referenced from `FlowTimeline.props.participants` rather than `children`.

```json
{
  "type": "SequenceParticipant",
  "props": {
    "label": "API Gateway",
    "kind": "boundary",
    "description": "Public edge service for auth traffic"
  }
}
```

| Prop           | Type                | Values                                                                 |
| -------------- | ------------------- | ---------------------------------------------------------------------- |
| `label`        | `string`            | Required. Visible participant label.                                   |
| `kind`         | `string`            | `participant`, `actor`, `boundary`, `control`, `entity`, `database`, `collections`, `queue` |
| `description`  | `string`            | Optional short description shown in the header card.                   |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file.                                                   |

## SequenceMessage

Ordered interaction between two participants.

```json
{
  "type": "SequenceMessage",
  "props": {
    "from": "gateway",
    "to": "auth",
    "label": "Validate credentials",
    "kind": "sync",
    "activate": ["auth"]
  }
}
```

| Prop           | Type                | Values                    |
| -------------- | ------------------- | ------------------------- |
| `from`         | `string`            | Required. Source participant id. |
| `to`           | `string`            | Required. Target participant id. |
| `label`        | `string`            | Required. Visible message label. |
| `description`  | `string`            | Optional narrative detail. |
| `kind`         | `string`            | `sync`, `async`, `return` |
| `activate`     | `string[]`          | Participants whose activation bars should start after this message. |
| `deactivate`   | `string[]`          | Participants whose activation bars should end on this message. |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file. |

## SequenceNote

Inline note attached to one or two participants.

```json
{
  "type": "SequenceNote",
  "props": {
    "label": "Rejected login",
    "body": "Return a generic auth error and increment fraud signals.",
    "placement": "right-of",
    "participants": ["gateway"]
  }
}
```

| Prop           | Type                | Values                         |
| -------------- | ------------------- | ------------------------------ |
| `label`        | `string`            | Optional note title.           |
| `body`         | `string`            | Required. Note body content.   |
| `placement`    | `string`            | `left-of`, `right-of`, `over`  |
| `participants` | `string[]`          | Required. One or two participant ids. |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file.           |

## SequenceGroup

Grouping container for Mermaid-like sequence blocks such as `alt`, `loop`, and `opt`.

```json
{
  "type": "SequenceGroup",
  "props": {
    "kind": "alt",
    "label": "Credentials valid?"
  },
  "children": ["branch-valid", "branch-invalid"]
}
```

| Prop           | Type                | Values             |
| -------------- | ------------------- | ------------------ |
| `label`        | `string`            | Required. Group label. |
| `kind`         | `string`            | `alt`, `loop`, `opt` |
| `description`  | `string`            | Optional context under the group badge. |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file. |

`SequenceGroup.children` should point to `SequenceBranch` elements.

## SequenceBranch

Ordered branch nested inside a `SequenceGroup`.

```json
{
  "type": "SequenceBranch",
  "props": {
    "label": "Valid"
  },
  "children": ["msg-validate", "msg-session"]
}
```

| Prop           | Type                | Description |
| -------------- | ------------------- | ----------- |
| `label`        | `string`            | Optional branch label. Strongly recommended for `alt`. |
| `description`  | `string`            | Optional branch description. |
| `sourceAnchor` | `SourceAnchorInput` | Link to source file. |

## ArchitectureNode

Infrastructure and service nodes for architecture mode. Supports the richest metadata set.

```json
{
  "type": "ArchitectureNode",
  "props": {
    "label": "API Gateway",
    "kind": "gateway",
    "zone": "edge",
    "tier": "edge",
    "status": "active",
    "technology": "Bun.serve",
    "owner": "Platform Team",
    "description": "Terminates TLS and routes to internal services",
    "responsibilities": ["Enforce auth policies", "Rate-limit traffic"],
    "capabilities": ["jwt-validation", "rate-limit"],
    "interfaces": [
      { "name": "Public HTTPS", "protocol": "HTTPS", "direction": "inbound", "auth": "JWT" }
    ],
    "security": { "auth": "JWT", "encryption": "TLS 1.3", "pii": false },
    "operations": { "slo": "99.95%", "alert": "5xx-rate > 1%", "runbook": "docs/gateway.md" },
    "links": [{ "label": "Dashboard", "href": "https://example.com/dashboard" }]
  }
}
```

| Prop               | Type                      | Values                                                                    |
| ------------------ | ------------------------- | ------------------------------------------------------------------------- |
| `label`            | `string`                  | Required. Node title.                                                     |
| `kind`             | `string`                  | `service`, `database`, `queue`, `cache`, `gateway`, `external`, `compute` |
| `tier`             | `string`                  | `edge`, `application`, `integration`, `data`, `platform`, `external`      |
| `status`           | `string`                  | `planned`, `active`, `degraded`, `retired`                                |
| `zone`             | `string`                  | ID of a zone defined in `FlowTimeline.props.zones`.                       |
| `technology`       | `string`                  | Technology stack.                                                         |
| `runtime`          | `string`                  | Runtime environment.                                                      |
| `owner`            | `string`                  | Owning team.                                                              |
| `description`      | `string`                  | Node description.                                                         |
| `responsibilities` | `string[]`                | What this service is responsible for.                                     |
| `capabilities`     | `string[]`                | Technical capabilities.                                                   |
| `interfaces`       | `ArchitectureInterface[]` | Inbound/outbound interface contracts.                                     |
| `data`             | `ArchitectureDataAsset[]` | Data assets managed by this node.                                         |
| `security`         | `ArchitectureSecurity`    | Auth, encryption, PII flag.                                               |
| `operations`       | `ArchitectureOperations`  | SLO, alerts, runbook links.                                               |
| `links`            | `ArchitectureLink[]`      | External dashboards or docs.                                              |
| `sourceAnchor`     | `SourceAnchorInput`       | Link to source file.                                                      |
| `transitions`      | `EdgeTransition[]`        | Outgoing edge metadata.                                                   |

## SourceAnchor

All node types accept a `sourceAnchor` prop. It can be a compact string or a structured object.

```json
// Compact string
"sourceAnchor": "src/auth/validate.ts:42"

// Structured object
"sourceAnchor": {
  "path": "src/auth/validate.ts",
  "line": 42,
  "label": "Validate handler",
  "href": "https://github.com/org/repo/blob/main/src/auth/validate.ts#L42"
}
```
