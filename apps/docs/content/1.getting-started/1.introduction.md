# Introduction

**Flaier** is a Vue component library that renders JSON flow specs as interactive diagrams. It is built for AI-generated codebase storytelling, where an LLM produces a structured spec describing how code flows through a system, and Flaier turns that spec into a visual, navigable timeline or architecture map.

## What problem it solves

Reading source code alone is hard. Understanding _how_ data moves through a system — across handlers, queues, databases, and external services — requires mental simulation that is slow and error-prone.

Flaier bridges the gap: AI agents produce a declarative spec (a JSON file) that describes each step, decision, and data transformation. Flaier renders that spec so humans can navigate it visually, step through it like a slideshow, or explore it as a topology map.

## Core concepts

| Concept            | Description                                                                                                                    |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------ |
| **Flow spec**      | A JSON file describing nodes (steps) and their edges (transitions).                                                            |
| **Manifest**       | A JSON index of multiple flow specs with metadata for each.                                                                    |
| **Node type**      | The visual/semantic role of a step — `trigger`, `code`, `decision`, `payload`, `error`, `description`, `link`, `architecture`. |
| **Rendering mode** | `narrative` for timeline playback, `architecture` for static topology.                                                         |

| `apps/docs`           | Docus site that documents and embeds the renderer.                      |
| `apps/slides`         | Slidev example deck for presentation-style embeds.                      |

## How AI agents use it

The spec format is intentionally flat and explicit so LLMs produce it reliably. Each element in `elements` has a `type`, a `props` object, and an optional `children` array. There is no nesting — the graph is encoded as parent → child references.

Agent harnesses can generate each `*.flow.json` in separate sessions and append or update the manifest incrementally, so large codebases can be narrated piecemeal.
